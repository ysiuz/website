Building a comparison experience that helps students and families compare programs and decide what’s worth applying to.

❶ Background

Product Context

Helping families navigate summer education options

BestSummerPrograms centralizes information about high school summer programs to help students and families navigate their complexity and cost.

Problem

Users couldn’t easily compare programs without leaving the site

Comparing program details like cost and location is an essential part of making informed application decisions, but without a native comparison feature, users had to juggle tabs or take notes externally for context.

Goal

Make it easier to compare programs while browsing

Our focus was to support program comparison within the platform, so users can quickly put program info in context while they browse and decide which ones to save.

Solution

Compare as you browse, accessible across the site

Users can mark programs for comparison from multiple entry points, adding them to a persistent taskbar as they browse. When ready, they can compare selected programs side by side, evaluating them based on their individual needs.

General compare flow.

1. Read program details.

2. Select program for comparison.

3. Confirm comparison with selected programs in taskbar.

4. Program comparison modal.

❷ Desktop Experience

Checkbox Entry Point

Click Compare Program to get started

I added a Compare checkbox to the action section on program pages, making comparisons accessible sitewide.

Surfacing actions on hover to reduce friction

User testing revealed frustration with needing to open each program page to save or compare programs. We reduced this friction by exposing Save and Compare actions on hover, prior to page entry.

Similar Program Hover: Leveraging similarity to prompt action

Because the Similar Programs bar surfaces contextually related programs, extending the hover pattern here allows users to compare high-relevance options without leaving the page

Compare Taskbar

Keeping selected programs in view as you browse

The taskbar provides persistent visual feedback for selected programs, helping users stay oriented as they browse and add find more programs to compare.

Collapsible, yet still informative and interactive

When collapsed, the taskbar shrinks to a minimal strip, offering links to the program pages and the ability to remove your selection on hover.

Compare Modal

See program details side by side

The compare modal lays out each program’s core information—badges, affiliation, selectivity, cost, location, and more—so users can compare them in detail and decide which programs to pursue.

Save programs or subtract from the modal

From the modal, users can add programs to their main list, refine their comparison by removing options, and clear their comparison panel once they’re done.

❸ Mobile Experience

Overview

The same experience but more compact

Mobile comparison follows the same structure as desktop, reducing the comparison cap from three to two programs to balance consistency with mobile screen constraints.

Keeping patterns consistent

A more minimal mobile comparison was explored but ultimately discarded due to its divergence from desktop patterns. A denser, two-column approach preserves cross-platform consistency, and our transparent information approach.

❸ Results

Takeaway

Prototyping with code is sick :^)

Because of Atlas' niche context and goal, I couldn't rely just on creative software to design my map. I needed to actually code to find the interactions and visual feedback that felt most natural.

Next Steps

Building a more robust onboarding and takeaway

The most common feedback I've received about Atlas is that it's a bit daunting and could use a friendly onboarding. In tandem, it will be great to work on what things users can tangibly takeaway from the experience, like potentially a receipt of their algorithmic profile, or shareable png's.
